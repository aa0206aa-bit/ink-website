日期：2026-06-12

## 1. 專案概述

「序序」是販售藝術流動墨（alcohol ink 類）的品牌。本網站是品牌門面：以**可互動的墨水流體體驗**讓訪客親手感受墨水技法，再以作品與產品說服購買，購買行為外連至既有賣場。

- 受眾：對流動墨藝術有興趣的消費者（手作、藝術愛好者）
- 商業模式：展示型網站＋外連購買（無站內金流）
- 語言：繁體中文

## 3. 頁面結構（單頁，由上而下）

1. **導覽列** — 「序序」字標＋錨點連結（體驗・墨色・作品・關於）
2. **Hero** — 品牌名＋標語＋「往下捲親手玩墨」引導。背景為 CSS 靜態墨暈（radial-gradient＋微緩動畫），**不放第二個 WebGL context**，互動聚焦於體驗區
3. **互動體驗區（核心）** — 技法工具列＋主畫布＋色盤＋清除鍵
4. **墨色（產品區）** — 每色一卡：色票、色號名、「帶它回家」外連按鈕；與色盤同一份資料來源
5. **作品展示** — 不規則磚牆排列（CSS grid），佔位視覺先頂位
6. **關於序序** — 三五句品牌故事（佔位文案）
7. **Footer** — IG／蝦皮／聯絡（佔位連結）＋版權

## 4. 互動體驗區設計（核心功能）

### 4.1 三種技法

| 技法 | 操作（桌機／手機通用） | 流體輸入 |
|------|------------------------|----------|
| 滴墨 | 點擊／輕觸畫布 | 注入染料＋微弱徑向速度脈衝 → 自然暈開 |
| 吹墨 | 按住拖曳 | 沿軌跡注入強方向速度、窄半徑 → 觸鬚狀紋路 |
| 傾斜 | 拖曳方向＝傾斜方向 | 拖曳向量轉為全域外力，整片墨緩慢流動，鬆手漸停 |

### 4.2 流體引擎（`fluid.js`）

- 標準 stable fluids 演算法：advection（平流）→ divergence（散度）→ pressure solve（壓力求解，Jacobi 迭代）→ gradient subtract（梯度修正），染料場獨立 advect
- Three.js 職責：WebGL context、`WebGLRenderTarget` ping-pong（雙緩衝）、`ShaderMaterial` 管理、全螢幕 quad 渲染
- 對外介面（頁面只透過這層溝通，不碰內部）：
  - `new InkSimulation(canvasElement, options)`
  - `setColor(hex)` — 設定當前墨色
  - `setMode('drop' | 'blow' | 'tilt')` — 切換技法
  - `clear()` — 清空畫布
  - `pause()` / `resume()` — 暫停／恢復模擬
  - 指標事件由引擎內部監聽 canvas 處理

### 4.3 色盤連動產品

- 色盤色球＝販售色號，點選作畫色即切換
- 色球與產品卡渲染自同一份 `INKS` 資料（見 §6.2）

## 7. 效能策略

- 模擬解析度與染料解析度分離：速度場 128–256 格；染料桌機 1024／手機 512
- `devicePixelRatio` 上限 2
- IntersectionObserver：畫布捲出視口 → `pause()`；`visibilitychange` 切背景分頁 → `pause()`
- `prefers-reduced-motion`：停用 hero 自動動畫；互動畫布因屬用戶主動操作，保留但停用閒置漂流

## 8. 錯誤處理

| 情境 | 處理 |
|------|------|
| WebGL 不可用 | 體驗區顯示靜態墨暈視覺＋文案提示，其餘區塊正常。不寫 Canvas 2D 備援引擎（工程量翻倍，>97% 瀏覽器支援 WebGL，不值得） |
| WebGL context lost（手機切 app 常見） | 監聽 `webglcontextlost` / `webglcontextrestored`，自動重建模擬 |
| 字型 CDN 失效 | `font-display: swap`＋系統字型 fallback，無感降級 |

## 9. 觸控與捲動

- 畫布元素 `touch-action: none`，互動時不觸發頁面捲動
- 畫布不滿版寬（兩側留邊），手機用戶可從畫布外捲動頁面
